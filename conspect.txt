1. git pull - стягиваем изменения из оригинальной папки (из репозитория) в локальную (на комп) - актуализируем внесенные кем-то изменения

2. git checkout -b название ветки - создаем новую ветку

3. git branch - проверить, в какой ветке находимся

после этого вносим изменения

4. git status - проверка статуса (какие файлы изменены или добавлены, что нужно добавить в commit)

5. git add . - подготовить к commit'у все файлы из папки, которые были изменены
   git add .название файла без расширения - подготовить к commit'у конкретный файл из папки

6. git status - убедиться, что нужные файлы были подготовлены (становятся зелеными)
