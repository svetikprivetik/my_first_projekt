1. git pull
 - стягиваем изменения из оригинальной папки (из репозитория) в локальную (на комп) - актуализируем внесенные кем-то изменения

2. git checkout -b название ветки
 - создаем новую ветку

3. git branch
 - проверить, в какой ветке находимся

после этого вносим изменения

4. git status
 - проверка статуса (какие файлы изменены или добавлены, что нужно добавить в commit)

5. git add .
 - подготовить к commit'у все файлы из папки, которые были изменены
 или
   git add .название файла без расширения
 - подготовить к commit'у конкретный файл из папки

6. git status
 - убедиться, что нужные файлы были подготовлены (становятся зелеными)

7. git commit -m'описание изменения'
 - добавляем комментарий об изменениях

8. git ststus
 - убеждаемся, что новых изменений нет

9. git pull origin master
 - в текущую ветку вносим изменения из удаленного master'а, если кто-то внес туда изменения

10. git push
 - отправляем наши изменения. Если предлагает путь (в первый раз) - скопировать и ввести вместо команды

11. переходим в Git в браузере, в наш проект, вкладка Code. Там видим на желтом поле данные о том, когда был запушен коммит

12. Жмем зеленую кнопку Compare & pull request --> Create pull request

Теперь есть две ветки проекта с разными версиями проекта

Чтобы добавить наши изменения в основную ветку (в master)

13. Переходим в новую ветку (которую запушили), нажимаем на значок с ветками (под зеленой кнопкой Code) затем зеленую кнопу Merge pull request, затем Confirm merge

Теперь все изменения есть в master'е

14. Чтобы открыть созданную страничку в браузере:
Setting --> Pages --> открываем голубую ссылку на зеленом фоне

15. Чтобы сдать домашку, отправляем эту ссылку и ссылку на репозиторий (папка Master, вкладка Code)
